display()

public function display(array $pagination_data): void

Description

Renders pagination navigation controls with an optional "Showing X to Y of Z records" statement. This method processes the provided pagination configuration through the model layer for validation and data preparation, then outputs the complete pagination HTML including First, Previous, numbered page links, Next, and Last navigation controls.

The method intelligently handles several scenarios:

  • If total_rows does not exceed limit, no pagination markup is output — pagination is unnecessary for a single page of results.
  • If pagination_root is not provided, the module auto-detects the base URL from the current request.
  • If page_num_segment is not provided, the module auto-detects by checking if the last URL segment is numeric.
  • The current page is automatically extracted from the URL and validated against the total number of pages.
  • Page number links are intelligently centered around the current page, displaying a configurable maximum number of links.

Parameters

Parameter Type Description
$pagination_data array An associative array containing pagination configuration. See the $pagination_data Keys section below for all available keys.

$pagination_data Keys

Key Type Description Default Required
total_rows int The total number of records in the dataset being paginated. Used to calculate the total number of pages and to determine if pagination is necessary. N/A Yes
limit int The number of records to display per page. Used in conjunction with total_rows to calculate pagination. Must be a positive integer. N/A Yes
record_name_plural string The plural noun for the records being paginated (e.g., 'products', 'articles', 'tasks'). Used in the default showing statement and for user-friendly messaging. N/A Yes
page_num_segment int|null The URL segment position (1-indexed) that contains the current page number. For example, in the URL /tasks/manage/2, the page number is in segment 3. When null, the module auto-detects by checking if the last URL segment is numeric. null No
pagination_root string|null The base URL path for pagination links (without the page number). For example: 'tasks/manage' or 'products/browse'. When null, the module auto-detects from the current URL by removing any trailing numeric segment and query strings. Should be specified as a relative path; BASE_URL is prepended automatically by the build_link() method. null No
include_showing_statement bool Whether to display a statement such as "Showing 1 to 10 of 25 tasks" above the pagination links. The statement is rendered using either the default English format or a custom template provided via the showing_statement key. false No
showing_statement string|null A custom template for the showing statement. Use placeholders {start}, {end}, and {total} to insert dynamic values. For example: 'Viewing {start}-{end} of {total} products'. When null, the default English format is used: 'Showing {start} to {end} of {total} record_name_plural.' null No
num_links_per_page int The maximum number of individual page number links to display at once. The current page is centered within this range when possible. For example, with a value of 7, if you are on page 5, you would see pages 2, 3, 4, 5, 6, 7, 8. Minimum recommended value is 3. 7 No
settings array Custom HTML wrapping and labels for pagination elements. Allows you to control the markup structure and link text. See the Custom Settings section below for available keys. Default settings array No
include_css bool Whether to include the default pagination CSS styling inline in the output. false No

The settings key accepts an array of custom HTML wrappers and labels. All of the following keys are optional and will use default values if not provided:

cur_link_open Opening HTML tag for the current (active) page link. '<a class="active">'
cur_link_close Closing HTML tag for the current (active) page link. '</a>'

num_link_close Closing HTML tag for numbered page links. '' (empty string)

last_link_close Closing HTML tag for the "Last" link. '' (empty string)

next_link_close Closing HTML tag for the "Next" link. '' (empty string)

Return Value

Type Description
void This method does not return a value. It outputs the pagination HTML directly to the response buffer via echo.

Example #1: Basic Usage with Auto-detection

Minimal configuration using auto-detection of page number and pagination root. Ideal for simple implementations.

[code=php]<?php // In your controller public function manage(): void { $this->trongate_security->make_sure_allowed(); $limit = 20; $offset = ($this->get_current_page() - 1) * $limit; $rows = $this->model->fetch_records($limit, $offset); $data = [ 'rows' => $rows, 'pagination_data' => [ 'total_rows' => $this->model->count_all(), 'limit' => $limit, 'record_name_plural' => 'tasks', 'include_showing_statement' => true ], 'view_module' => 'tasks', 'view_file' => 'manage' ]; $this->templates->admin($data); } ?>[/code]

In your view file:

[code=php]<?php echo Modules::run('pagination/display', $pagination_data); ?>[/code]

Example #2: Explicit Configuration with Custom Settings

Providing explicit pagination root, page segment, and custom HTML styling.

[code=php]<?php $pagination_data = [ 'total_rows' => 487, 'limit' => 20, 'record_name_plural' => 'products', 'page_num_segment' => 3, 'pagination_root' => 'products/browse', 'include_showing_statement' => true, 'include_css' => true, 'settings' => [ 'pagination_open' => '<nav class="pagination-controls" role="navigation" aria-label="pagination">', 'pagination_close' => '</nav>', 'cur_link_open' => '<span class="page-number active">', 'cur_link_close' => '</span>', 'first_link' => '« First', 'last_link' => 'Last »', 'prev_link' => '← Previous', 'next_link' => 'Next →' ] ]; Modules::run('pagination/display', $pagination_data); ?>[/code]

Example #3: Custom Showing Statement with Custom Link Count

Using a custom template for the showing statement and controlling the number of page links displayed.

[code=php]<?php $pagination_data = [ 'total_rows' => 156, 'limit' => 10, 'record_name_plural' => 'articles', 'pagination_root' => 'blog/articles', 'include_showing_statement' => true, 'showing_statement' => 'Items {start}–{end} of {total}', 'num_links_per_page' => 5 ]; Modules::run('pagination/display', $pagination_data); ?>[/code]

Example #4: Per-Page Selector Integration (Like the Tasks Module)

Full implementation with user-configurable records per page, as demonstrated in the Tasks module.

[code=php]<?php // In your controller private int $default_limit = 20; private array $per_page_options = [10, 20, 50, 100]; public function manage(): void { $this->trongate_security->make_sure_allowed(); // Get limit from session (user preference) $limit = $this->get_limit(); // Calculate offset based on current page $page_num = segment(3, 'int'); $offset = ($page_num > 1) ? ($page_num - 1) * $limit : 0; // Fetch records $rows = $this->model->fetch_records($limit, $offset); // Prepare pagination data $pagination_data = [ 'total_rows' => $this->model->count_all(), 'page_num_segment' => 3, 'limit' => $limit, 'pagination_root' => 'tasks/manage', 'record_name_plural' => 'tasks', 'include_showing_statement' => true ]; $data = [ 'rows' => $rows, 'pagination_data' => $pagination_data, 'per_page_options' => array_keys($this->per_page_options), 'selected_per_page' => $_SESSION['selected_per_page'] ?? 1, 'view_module' => 'tasks', 'view_file' => 'manage' ]; $this->templates->admin($data); } public function set_per_page(): void { $this->trongate_security->make_sure_allowed(); $_SESSION['selected_per_page'] = segment(3, 'int'); redirect('tasks/manage'); } private function get_limit(): int { $per_page_options = [10, 20, 50, 100]; if (isset($_SESSION['selected_per_page'])) { return $per_page_options[$_SESSION['selected_per_page']]; } return $this->default_limit; } ?>[/code]

In your view file:

[code=php]<?php echo Modules::run('pagination/display', $pagination_data); ?> <select onchange="window.location.href = '<?= BASE_URL ?>tasks/set_per_page/' + this.value"> <?php foreach($per_page_options as $index => $option): ?> <option value="<?= $index ?>" <?= ($selected_per_page == $index) ? 'selected' : '' ?>> <?= $option ?> per page </option> <?php endforeach; ?> </select>[/code]

Notes

  • Early Return: If total_rows ≤ limit, the method returns without output. No pagination is needed for a single page of results.
  • URL Structure: Page 1 uses the clean pagination_root URL; subsequent pages append the page number (e.g., /tasks/manage/2, /tasks/manage/3).
  • Current Page Validation: If the current page exceeds the total number of pages (e.g., user manually edits the URL), the current page is clamped to the maximum page.
  • Security: All URLs and ARIA labels are properly escaped using htmlspecialchars() with ENT_QUOTES.
  • Accessibility: All navigation links include aria-label attributes for screen readers.
  • Link Centering: The num_links_per_page parameter intelligently centers the current page within the link range. When near the beginning or end of the dataset, the range adjusts to always display the maximum number of links (when possible).
