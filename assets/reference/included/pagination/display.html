display()

public function display(array $pagination_data): void

Description

Renders pagination navigation controls with optional "Showing" statement. This method processes the pagination configuration through the model layer for validation, then outputs the complete pagination HTML. The method handles all link types — First, Previous, numbered pages, Next, and Last — with automatically detected or explicitly specified URL roots.

If the total number of records does not exceed the limit per page, the method returns early without rendering any output — no unnecessary markup.

Parameters

Parameter Type Description Default Required
$pagination_data array An associative array of pagination configuration (see below).

$pagination_data Keys

Key Type Description Default Required
total_rows int The total number of records being paginated. N/A Yes
limit int The number of records to display per page. N/A Yes
record_name_plural string The plural name for the records (e.g., 'products', 'articles') used in the showing statement. N/A Yes
include_showing_statement bool Whether to display a "Showing X to Y of Z records" statement above the pagination links. false No
page_num_segment int|null The URL segment containing the current page number. When null, the module auto-detects by checking if the last URL segment is numeric. null No
pagination_root string|null The base URL path for pagination links. When null, the module auto-detects from the current URL. null No
settings array Custom HTML wrapping for pagination links. Accepts keys such as pagination_open, pagination_close, cur_link_open, cur_link_close, num_link_open, num_link_close, first_link, last_link, prev_link, next_link, and their corresponding *_aria_label keys. [] No
include_css bool Whether to include default pagination CSS styling. false No
showing_statement string|null A custom showing statement template. Use {start}, {end}, and {total} as placeholders for dynamic values. null No
num_links_per_page int The maximum number of page number links to display at once. 7 No

Return Value

Type Description
void This method does not return a value. It outputs the pagination HTML directly.

Example #1

Controller method demonstrating pagination setup for a tasks module. The pagination data is prepared in a private helper method and passed to the view. Pagination is rendered in the view using Modules::run().

[code=php]class Tasks extends Trongate { private int $default_limit = 20; private array $per_page_options = [10, 20, 50, 100]; public function manage(): void { $this->trongate_security->make_sure_allowed(); $limit = $this->get_limit(); $rows = $this->model->fetch_records($limit, $this->get_offset()); $data = [ 'rows' => $rows, 'pagination_data' => $this->get_pagination_data($limit), 'view_module' => 'tasks', 'view_file' => 'manage' ]; $this->templates->admin($data); } private function get_pagination_data(int $limit): array { return [ 'total_rows' => $this->model->count_all(), 'page_num_segment' => 3, 'limit' => $limit, 'pagination_root' => 'tasks/manage', 'record_name_plural' => 'tasks', 'include_showing_statement' => true ]; } private function get_limit(): int { // Returns records-per-page from session or default return $_SESSION['selected_per_page'] ?? $this->default_limit; } private function get_offset(): int { $page_num = segment(3, 'int'); return ($page_num > 1) ? ($page_num - 1) * $this->get_limit() : 0; } }[/code]

Example #2

View file (manage.php) showing how pagination is rendered. The pagination controls appear both above and below the records table. The include_showing_statement key is removed from the second call to avoid duplicate output.

[code=php]<h1>Manage Tasks</h1> <?php echo Modules::run('pagination/display', $pagination_data); ?> <table class="records-table"> <thead> <tr> <th>Task Title</th> <th>Complete</th> <th>Action</th> </tr> </thead> <tbody> <?php foreach($rows as $row) { ?> <tr> <td><?= out($row->task_title) ?></td> <td><?= out($row->complete_formatted) ?></td> <td><?= anchor('tasks/show/'.$row->id, 'View') ?></td> </tr> <?php } ?> </tbody> </table> <?php // Second pagination block below table (without showing statement) unset($pagination_data['include_showing_statement']); echo Modules::run('pagination/display', $pagination_data); ?>[/code]

Example #3

Customising pagination output with a showing statement that uses placeholders. The showing_statement key accepts {start}, {end}, and {total} as dynamic placeholders.

[code=php]$pagination_data = [ 'total_rows' => $total_rows, 'limit' => 15, 'page_num_segment' => 3, 'pagination_root' => 'products/manage', 'record_name_plural' => 'products', 'include_showing_statement' => true, 'showing_statement' => 'Viewing {start} to {end} of {total} products' ]; echo Modules::run('pagination/display', $pagination_data);[/code]
